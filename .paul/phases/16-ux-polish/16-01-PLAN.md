<objective>
## Goal
Add drag-and-drop file loading onto the waveform zone and an elapsed-time counter
in the status label during long separation/remix operations.

## Purpose
Loading a file via a file-chooser dialog is slow on WSL2. Drag & drop is the
standard DAW workflow. Elapsed time removes the "is it frozen?" anxiety during
Demucs separation (can take 30–120s).

## Output
- `WaveformDisplay` (or editor) accepts drag-and-drop of audio files; triggers same
  flow as `loadFile()` (thumbnail + analysis)
- During `onClick_Play`, status label updates every second:
  "Separating stems... 14s" → "Remixing... 3s" → "Done - click Save to export."
- Timer stops on completion or error
</objective>

    ```cpp
    void filesDropped(const juce::StringArray& files, int x, int y) override;
    ```

    In AutoRemixEditor.cpp, implement both methods:

    ```cpp
    static const juce::StringArray kAudioExts {
        ".wav", ".aif", ".aiff", ".mp3", ".flac", ".ogg", ".m4a"
    };

    bool AutoRemixAudioProcessorEditor::isInterestedInFileDrag(
        const juce::StringArray& files)
    {
        if (files.isEmpty()) return false;
        auto ext = juce::File(files[0]).getFileExtension().toLowerCase();
        return kAudioExts.contains(ext);
    }

    void AutoRemixAudioProcessorEditor::filesDropped(
        const juce::StringArray& files, int /*x*/, int /*y*/)
    {
        if (files.isEmpty()) return;
        juce::File f(files[0]);
        if (!f.existsAsFile()) return;
        auto ext = f.getFileExtension().toLowerCase();
        if (!kAudioExts.contains(ext)) return;

        file_path_ = f.getFullPathName();
        file_lbl.setText(f.getFileName(), juce::dontSendNotification);
        output_path_ = {};
        save_btn.setEnabled(false);
        thumbnail_.setSource(new juce::FileInputSource(f));
        waveform_display_.sourceChanged();

        std::thread([this, path = file_path_]() {
            auto info = audioProcessor.getBridge().analyzeFile(path);
            float bpm = info.valid() ? info.bpm : 120.0f;
            juce::String key = info.valid() ? juce::String(info.key) : "";
            juce::MessageManager::callAsync([this, bpm, key]() {
                detected_bpm_ = bpm;
                detected_key_ = key;
                juce::String txt = juce::String(bpm, 1) + " BPM";
                txt += key.isNotEmpty() ? ("  |  " + key) : "  |  ?";
                analysis_lbl_.setText(txt, juce::dontSendNotification);
                if (bpm >= 40.0f && bpm <= 200.0f)
                    tempo_slider_.setValue((double)bpm, juce::dontSendNotification);
            });
        }).detach();
    }
    ```

    Note: this duplicates the file-loading logic from `loadFile()`. That is intentional —
    do NOT refactor into a helper (out of scope; risk of regression).

<task type="auto">
  <name>Task 2: Elapsed-time counter in status label</name>
  <files>plugin/include/PluginEditor.h, plugin/src/AutoRemixEditor.cpp</files>
  <action>
    In PluginEditor.h, add a timer member and elapsed counter:
    ```cpp
    juce::Timer*   elapsed_timer_ = nullptr;  // owned by heap; stopped and deleted on done
    std::atomic<int> elapsed_secs_{0};
    std::atomic<bool> in_remix_phase_{false};
    ```

    Use a lambda timer pattern — JUCE timers must run on the message thread.
    Use `juce::Timer` subclass approach inline. Instead of a full subclass,
    use `juce::MessageManager::callAsync` scheduled recursively.

    Better approach: add a `juce::Timer`-derived inner struct in the .cpp:

    In onClick_Play(), before the thread launch, start a repeating timer:
    ```cpp
    // start elapsed counter
    elapsed_secs_.store(0);
    in_remix_phase_.store(false);
    if (elapsed_timer_) { elapsed_timer_->stopTimer(); delete elapsed_timer_; }
    struct ElapsedTimer : juce::Timer {
        AutoRemixAudioProcessorEditor& ed;
        explicit ElapsedTimer(AutoRemixAudioProcessorEditor& e) : ed(e) {}
        void timerCallback() override {
            int s = ed.elapsed_secs_.fetch_add(1) + 1;
            bool remix = ed.in_remix_phase_.load();
            juce::String phase = remix ? "Remixing... " : "Separating stems... ";
            ed.status_lbl.setText(phase + juce::String(s) + "s",
                                  juce::dontSendNotification);
        }
    };
    elapsed_timer_ = new ElapsedTimer(*this);
    elapsed_timer_->startTimer(1000);
    ```

    In the background thread, when switching from separation to remix:
    ```cpp
    juce::MessageManager::callAsync([this] {
        elapsed_secs_.store(0);
        in_remix_phase_.store(true);
    });
    ```

    On completion or error (in the callAsync final block), stop and delete timer:
    ```cpp
    if (elapsed_timer_) { elapsed_timer_->stopTimer(); delete elapsed_timer_; elapsed_timer_ = nullptr; }
    ```

    In destructor, stop timer:
    ```cpp
    if (elapsed_timer_) { elapsed_timer_->stopTimer(); delete elapsed_timer_; }
    ```

    Member declarations in PluginEditor.h — add after `analysis_lbl_`:
    ```cpp
    juce::Timer*     elapsed_timer_ = nullptr;
    std::atomic<int> elapsed_secs_{0};
    std::atomic<bool> in_remix_phase_{false};
    ```
