<objective>

## Goal
Add tooltips to unlabeled controls (health dot, separator combo, M/S buttons)
and unit suffixes to numeric sliders (`×`, `BPM`, `Hz`, `semi`). Removes the
"what does this do / what unit is this?" friction without adding visual chrome.

## Purpose
Users today see the `●` health dot, `M`/`S` toggles, and a separator combo
in the header with no labels. Sliders show `1.0` / `120` / `100` with no
units. Hovering with tooltips + suffixing the value boxes solves both
without redesigning anything.

## Output
- One `juce::TooltipWindow` instance in the editor (~750ms hover delay)
- Tooltips on: health dot, separator combo, M, S, play (▶/■) buttons
- Slider suffixes:
  - StemsReady stem volume sliders: `×`
  - ScreenMashup 8 stem gain sliders: `×`
  - ScreenMashup Tempo Mod: `×`
  - ScreenMashup HPF B: ` Hz`
  - ScreenMashup Master Pitch: ` semi`
  - ScreenMashup Target BPM + ScreenModeParams Target BPM: ` BPM`
  - ScreenModeParams Pitch slider: ` semi`
- No other widget or layout changes
</objective>

## Open Questions (revisit during APPLY)

- **Tooltip hover delay**: 750ms is JUCE default-ish — feels right for
  desktop. Drop to 500ms if 750 feels sluggish.
- **NoTextBox sliders**: ScreenStemsReady's stem volume sliders use
  NoTextBox style. setTextValueSuffix is a no-op there. Decision:
  leave them as-is (the visual position-of-thumb is the value
  indicator), or add a TextBoxRight + suffix. Default: leave as-is
  to avoid layout shift; revisit in a future polish if users complain.
- **Should the play button tooltip mention drag-to-DAW?** No — that's
  visually discoverable by trying. Tooltip stays scoped to the button's
  primary action.
