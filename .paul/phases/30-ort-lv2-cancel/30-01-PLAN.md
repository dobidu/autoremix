<objective>
## Goal
Upgrade ONNX Runtime from 1.17.0 to 1.21.0 and pin SHA256 for all 6 ORT packages
(3 CPU + 3 GPU) to eliminate supply-chain risk from empty `ORT_SHA ""` fields.

## Purpose
ORT 1.17.0 is from October 2023. ORT 1.21.0 brings improved CoreML EP performance on
Apple Silicon, updated CUDA 12 support, and DirectML EP fixes. SHA256 pinning prevents
silent tarball substitution attacks on the FetchContent download step — currently all GPU
packages have `ORT_SHA ""` which means any content served at that URL is accepted.

## Output
- `CMakeLists.txt`: `ORT_VERSION` → `1.21.0`; all URL paths updated; `ORT_SHA` fields
  populated with verified SHA256 for each of the 6 packages
</objective>

  <action>
    **Step 1 — version bump:**
    Change `set(ORT_VERSION "1.17.0")` → `set(ORT_VERSION "1.21.0")`.
    URLs are constructed from ORT_VERSION so they update automatically.

    **Step 2 — verify URL naming for 1.21.0:**
    ORT 1.21.0 packages follow the same naming convention as 1.17.0:
    - Linux CPU:    `onnxruntime-linux-x64-1.21.0.tgz`
    - Linux GPU:    `onnxruntime-linux-x64-gpu-1.21.0.tgz`
    - macOS:        `onnxruntime-osx-universal2-1.21.0.tgz`
    - Windows CPU:  `onnxruntime-win-x64-1.21.0.zip`
    - Windows GPU:  NuGet `Microsoft.ML.OnnxRuntime.DirectML/1.21.0`
    All URL templates already use `${ORT_VERSION}` — no explicit URL changes needed.

    **Step 3 — fetch SHA256 for each package:**
    Run the following to get the SHA256 of each tarball. Record the output.

    ```bash
    # Linux CPU
    curl -sL "https://github.com/microsoft/onnxruntime/releases/download/v1.21.0/onnxruntime-linux-x64-1.21.0.tgz" \
      | sha256sum | awk '{print $1}'

    # Linux GPU
    curl -sL "https://github.com/microsoft/onnxruntime/releases/download/v1.21.0/onnxruntime-linux-x64-gpu-1.21.0.tgz" \
      | sha256sum | awk '{print $1}'

    # macOS universal2
    curl -sL "https://github.com/microsoft/onnxruntime/releases/download/v1.21.0/onnxruntime-osx-universal2-1.21.0.tgz" \
      | sha256sum | awk '{print $1}'

    # Windows CPU
    curl -sL "https://github.com/microsoft/onnxruntime/releases/download/v1.21.0/onnxruntime-win-x64-1.21.0.zip" \
      | sha256sum | awk '{print $1}'

    # Windows GPU (DirectML NuGet)
    curl -sL "https://www.nuget.org/api/v2/package/Microsoft.ML.OnnxRuntime.DirectML/1.21.0" \
      | sha256sum | awk '{print $1}'
    ```

    Note: macOS and Windows GPU use the same ORT tarball as CPU (GPU EP is built-in for
    macOS universal2; DirectML NuGet is separate). There are 5 unique packages to pin.

    **Step 4 — populate ORT_SHA fields:**
    Replace each `set(ORT_SHA "")` with the computed SHA256 in the correct platform block.
    The existing CMakeLists.txt structure:
    - Linux CPU block:    `set(ORT_SHA "efc344d...")` — ALREADY pinned (update to 1.21.0 value)
    - Linux GPU block:    `set(ORT_SHA "")` — pin with fetched value
    - macOS block:        `set(ORT_SHA "")` — pin with fetched value
    - Windows CPU block:  `set(ORT_SHA "")` — pin with fetched value
    - Windows GPU block:  `set(ORT_SHA "")` — pin with fetched NuGet value

    **Step 5 — check API compatibility:**
    After building, grep build output for deprecation warnings related to:
    - `AppendExecutionProvider_CUDA` (old API — may be deprecated in 1.21, check for warning)
    - `OrtSessionOptionsAppendExecutionProvider_CoreML` (C API — should remain stable)
    If deprecation warnings appear, note them in SUMMARY but do NOT fix in this plan
    (API migration is a separate concern). Build must compile clean (no errors).

    Avoid: manually specifying URL paths — they're templated from ORT_VERSION.
    Avoid: changing any GPU EP code or session creation logic.
  </action>
