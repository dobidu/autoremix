<objective>
## Goal
Implement `NativeStructuredRemixEngine.h` — a new engine that calls
`analyze_structure()`, maps template arrangement slots to detected sections,
and assembles the output. Wire it into AutoRemixEditor and ship one example
structured preset.

## Purpose
This is the first consumer of the entire Phase 31 stack. It proves the loop:
template JSON → parsed StructureConfig → analyze_structure() → section mapping
→ audio output. The engine is a skeleton: arrangement + basic stem layer
application work end-to-end; scratch, build-up, and effects-per-section are
stubbed (Phase 34/35 scope). A working example preset validates the architecture.

- `plugin/src/AutoRemixEditor.cpp`: engine routed for engine id
  `"structured_remix"`
- `sidecar/server/presets/trap_banger_structured.json`: example structured preset

    Create new header-only file. Namespace: `autoremix::dsp::engines`.

    ```cpp
    #pragma once
    // NativeStructuredRemixEngine — Phase 31-04
    // Skeleton: analyze_structure() → map arrangement → assemble sections.
    // Scratch, build-up, per-section effects: stubs (Phase 34/35).
    ```

    Include: NativeAnalysis.h, NativePresetTypes.h, NativeRemixEngines.h,
    juce_audio_basics, juce_core.

    **`resolve_anchor()`** helper:
    ```cpp
    inline int resolve_anchor(const std::string& anchor,
                              const analysis::SongStructure& s,
                              int slot_idx)
    ```
    Maps anchor string to a bar index:
    - `"first"` → 0
    - `"second"` → s.num_bars / 4 (quarter-point)
    - `"last"` → s.num_bars - slot.length_bars (or max(0, ...))
    - `"energy_peak"` → s.energy_peaks.empty() ? 0 : s.bar_times index of first peak
    - `"bar:N"` → parse N from string
    - default → 0

    **`extract_bars()`** helper:
    ```cpp
    inline juce::AudioBuffer<float>
    extract_bars(const juce::AudioBuffer<float>& src, double sr,
                 const analysis::SongStructure& s,
                 int start_bar, int length_bars)
    ```
    - Clamp start_bar and length_bars to valid range
    - Compute start_sample = bar_times[start_bar] * sr
    - Compute end_sample = (start_bar + length_bars < num_bars)
        ? bar_times[start_bar + length_bars] * sr
        : src.getNumSamples()
    - Return AudioBuffer copy of that range

    **`apply_section_layer()`** helper:
    ```cpp
    inline void apply_section_layer(juce::AudioBuffer<float>& buf,
                                    const presets::SectionLayer& layer)
    ```
    Apply gain to the mix buffer: `buf.applyGain(layer.vocals * 0.25 + ...)`.
    Actually: since buf is already a mixed buffer (not per-stem), just apply
    a single loudness gain = average of the four stem gains. This is a
    simplification; per-stem layer application requires pre-mixed stems and
    comes in Phase 35.

    **`structured_remix()`** main function:
    ```cpp
    inline juce::AudioBuffer<float>
    structured_remix(const separators::NativeStems& stems, double sr,
                     const RemixParams& p,
                     const presets::NativeRemixPreset& preset)
    ```
    Algorithm:
    1. Mix stems equally → `mix_buf`
    2. If !preset.structure.enabled → fallback: `return chopped_and_screwed(stems, sr, p)`
    3. `auto song = analysis::analyze_structure(mix_buf_as_audio_buffer, sr)`
    4. Log: `juce::Logger::writeToLog("[StructuredRemix] sections: " + ...)`
    5. For each slot in `preset.structure.arrangement`:
       a. `int start = resolve_anchor(slot.source_anchor, song, i)`
       b. `int len = slot.length_bars > 0 ? slot.length_bars : 4`
       c. `auto section = extract_bars(mix_buf_as_audio_buffer, sr, song, start, len)`
       d. `apply_section_layer(section, slot.layer)`
       e. Loop `slot.loop_count` times: append section to output buffer
    6. Assemble all sections into one output buffer (concatenate)
    7. Apply global time-stretch/pitch-shift via TimePitchStretcher
       (use p.tempo_factor and p.pitch_shift_semi same as chopped_and_screwed)
    8. Apply reverb if p.reverb_mix > 0
    9. normalize_lufs to -14 LUFS
    10. Return output

    For step 2's `mix_buf_as_audio_buffer`: `mix_stems_equal()` returns
    `juce::AudioBuffer<float>`. Wrap as AudioBuffer to pass to analyze_structure.
    Actually analyze_structure already takes AudioBuffer — pass mix_buf directly.

    Avoid: storing SongStructure as a member — compute fresh per call.
    Avoid: implementing scratch/build-up yet — stub with TODO comment.
  </action>
