<objective>
## Goal
Define the `SongStructure` data type and implement the energy map + phrase grid
primitives — the foundational layer that every structure-aware feature in Phase 31+
will consume.

## Purpose
The template engine (Phase 31-03+) needs a runtime description of a song's
architecture to route audio intelligently. `SongStructure` is that description.
Without it, templates can only specify fixed parameters; with it, templates can
say "drop at energy_peak" or "loop the chorus phrase" and the engine knows
exactly which bar that maps to.

## Output
- `NativeAnalysis.h`: `SongStructure` struct + `energy_per_bar()` +
  `energy_peaks()` + `phrase_times()` helper.
  `analyze_structure()` entry point (partial — complete in 31-02).
</objective>

        // Rhythm
        double              bpm             = 0.0;
        std::vector<double> beat_times;     // seconds at each beat

        std::vector<int>    section_boundaries;

        int     num_bars         = 0;

    };

    Add three inline helper functions and the `analyze_structure()` entry point
    to the `autoremix::dsp::analysis` namespace.

    **`energy_per_bar()`** — compute RMS energy in each bar window:
    ```cpp
    inline std::vector<float>
    energy_per_bar(const juce::AudioBuffer<float>& audio, double sr,
                   const std::vector<double>& bar_times)
    ```
    Implementation:
    - For each bar interval [bar_times[i], bar_times[i+1]):
      - Compute RMS of the mono signal in that sample range
    - For the last bar: use [bar_times.back(), end_of_audio]
    - Normalize to [0, 1] by dividing by max RMS
    - Return vector<float> of size bar_times.size()

    **`energy_peaks()`** — find local maxima separated by at least min_sep bars:
    ```cpp
    inline std::vector<int>
    energy_peaks(const std::vector<float>& energy, int min_sep_bars = 4)
    ```
    Implementation:
    - Local maximum: energy[i] > energy[i-1] && energy[i] > energy[i+1]
    - Enforce min_sep_bars distance between consecutive peaks
    - Return indices

    **`phrase_times()`** — every N bars starting at bar 0:
    ```cpp
    inline std::vector<double>
    phrase_times(const std::vector<double>& bar_times, int phrase_bars = 4)
    ```
    Implementation: trivial — return bar_times[0], bar_times[4], bar_times[8]...

    **`analyze_structure()`** — entry point (partial, completed in 31-02):
    ```cpp
    inline SongStructure
    analyze_structure(const juce::AudioBuffer<float>& audio, double sr)
    {
        SongStructure s;
        s.duration_seconds = audio.getNumSamples() / sr;
        s.bpm              = detect_bpm(audio, sr);
        s.key              = detect_key(audio, sr);
        s.beat_times       = detect_beats(audio, sr);
        s.bar_times        = detect_bars(audio, sr, 4);
        s.phrase_times     = phrase_times(s.bar_times, 4);
        s.num_bars         = (int) s.bar_times.size();
        s.energy_per_bar   = energy_per_bar(audio, sr, s.bar_times);
        s.energy_peaks     = energy_peaks(s.energy_per_bar, 4);
        // brightness_per_bar, vocal_presence_per_bar, section_boundaries
        // filled in 31-02.
        return s;
    }
    ```

    Avoid: calling detect_bpm + detect_beats separately (detect_beats internally
    calls detect_bpm again — this double-computes STFT). Accept this cost for now;
    optimization deferred. Do NOT refactor detect_beats to avoid it (out of scope).
