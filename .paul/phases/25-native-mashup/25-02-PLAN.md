<objective>

## Goal
Port the v3 PresetLoader + MashupPresetLoader to native C++. Built-in
JSON files (9 remix + 8 mashup) get embedded in the binary via
`juce_add_binary_data`. User-supplied JSONs are read at startup from
the cross-platform user config dir.

## Purpose
Phase 27 wire-up will need to populate `ctx_.presets` and
`ctx_.mashup_presets` natively (replacing AudioBridge::getPresets and
::getMashupPresets). Closes Phase 25.

## Output
- New CMake `juce_add_binary_data` target `AutoRemixPresetsData` carrying:
  - All 9 JSONs from `sidecar/server/presets/` (excluding `schema.json`)
  - All 8 JSONs from `sidecar/server/mashup_presets/`
  Plugin links this target.
- `plugin/include/dsp/NativePresetTypes.h` — header-only structs:
    NativeRemixPreset { id, version, name, description, author, tags,
                        engine, params (RemixParams), stem_mix (StemMix),
                        effects (vector<EffectStep>) }
    NativeMashupPreset { id, version, name, description, author, tags,
                         stem_gains_a/b (unordered_map<string,float>),
                         target_bpm_mode/absolute, target_key_mode/absolute,
                         bpm_modifier, master_pitch_offset_semi,
                         master_reverb_mix, master_reverb_room,
                         highpass_b_hz }
- `plugin/include/dsp/NativePresetLoaders.h` — header-only loaders:
    `load_remix_presets() -> std::vector<NativeRemixPreset>`
    `load_mashup_presets() -> std::vector<NativeMashupPreset>`
  Both iterate `BinaryData::namedResourceList` for built-ins + scan
  the user config dir for `*.json`. User overrides built-in on id
  collision.
- Smoke include from `AutoRemixProcessor.cpp`. No call-sites yet.

              out.stem_mix.vocals = s.value("vocals", 1.0f);

      static inline NativeMashupPreset
      parse_mashup_preset(const nlohmann::json& j)
      {
          NativeMashupPreset out;
          out.id          = j.value("id",          std::string{});
          out.version     = j.value("version",     std::string{"1.0"});
          out.name        = j.value("name",        out.id);
          out.description = j.value("description", std::string{});
          out.author      = j.value("author",      std::string{"AutoRemix"});
          if (j.contains("tags") && j["tags"].is_array())
              for (auto& t : j["tags"]) out.tags.push_back(t.get<std::string>());
          auto read_gains = [](const nlohmann::json& src, std::unordered_map<std::string,float>& dst) {
              if (!src.is_object()) return;
              for (auto it = src.begin(); it != src.end(); ++it)
                  dst[it.key()] = it.value().get<float>();
          };
          if (j.contains("stem_gains_a")) read_gains(j["stem_gains_a"], out.stem_gains_a);
          if (j.contains("stem_gains_b")) read_gains(j["stem_gains_b"], out.stem_gains_b);
          out.target_bpm_mode      = j.value("target_bpm_mode",      std::string{"anchor_a"});
          out.target_bpm_absolute  = j.value("target_bpm_absolute",  0.0);
          out.target_key_mode      = j.value("target_key_mode",      std::string{"anchor_a"});
          if (j.contains("target_key_absolute") && j["target_key_absolute"].is_string())
              out.target_key_absolute = j["target_key_absolute"].get<std::string>();
          out.bpm_modifier             = j.value("bpm_modifier",             1.0f);
          out.master_pitch_offset_semi = j.value("master_pitch_offset_semi", 0.0f);
          out.master_reverb_mix        = j.value("master_reverb_mix",        0.0f);
          out.master_reverb_room       = j.value("master_reverb_room",       0.5f);
          out.highpass_b_hz            = j.value("highpass_b_hz",            0.0f);
          return out;
      }

    Loader helpers:

      // Returns true if the lowercased filename matches one of the
      // known mashup preset ids (drives the binary-data routing).
      static inline bool is_mashup_file(juce::StringRef name)
      {
          static const std::array<juce::StringRef, 8> kMashupIds = {
              "vocal_acapella","drum_swap","slowed_mashup","nightcore_mashup",
              "dub_echo","instrumental_layer","bass_swap","frankenstein"
          };
          // Binary-data namedResourceList strips dots; an input.json
          // becomes "input_json" in the table. Compare against the
          // expected "<id>_json" name.
          for (auto& id : kMashupIds) {
              if (name == juce::String(id.text) + "_json") return true;
          }
          return false;
      }

      static inline void
      load_builtins_from_binary(
          std::vector<NativeRemixPreset>& remix_out,
          std::vector<NativeMashupPreset>& mashup_out)
      {
          for (int i = 0; i < BinaryData::namedResourceListSize; ++i) {
              const char* name = BinaryData::namedResourceList[i];
              int size = 0;
              const char* data = BinaryData::getNamedResource(name, size);
              if (!data || size <= 0) continue;

              const std::string body(data, (size_t) size);
              try {
                  auto j = nlohmann::json::parse(body);
                  if (is_mashup_file(name))
                      mashup_out.push_back(parse_mashup_preset(j));
                  else
                      remix_out .push_back(parse_remix_preset(j));
              } catch (const std::exception& e) {
                  juce::Logger::writeToLog(
                      juce::String("[preset] failed to parse builtin ")
                      + name + ": " + e.what());
              }
          }
      }

      static inline void
      load_user_dir_remix(const juce::File& dir,
                          std::vector<NativeRemixPreset>& out)
      {
          if (!dir.isDirectory()) return;
          for (auto& f : dir.findChildFiles(juce::File::findFiles, false, "*.json")) {
              try {
                  auto body = f.loadFileAsString().toStdString();
                  out.push_back(parse_remix_preset(nlohmann::json::parse(body)));
              } catch (const std::exception& e) {
                  juce::Logger::writeToLog(
                      juce::String("[preset] failed to parse user ")
                      + f.getFullPathName() + ": " + e.what());
              }
          }
      }

      // Same for mashup (parse_mashup_preset)
      static inline void
      load_user_dir_mashup(const juce::File& dir,
                           std::vector<NativeMashupPreset>& out)
      { /* mirror load_user_dir_remix using parse_mashup_preset */ }

    Public:

      inline std::vector<NativeRemixPreset>
      load_remix_presets()
      {
          std::vector<NativeRemixPreset>  remix;
          std::vector<NativeMashupPreset> tmp_mashup;  // discarded here
          load_builtins_from_binary(remix, tmp_mashup);
          load_user_dir_remix(user_remix_dir(), remix);
          // De-dup by id, user wins (later wins)
          std::unordered_map<std::string, size_t> seen;
          for (size_t i = 0; i < remix.size(); ++i) seen[remix[i].id] = i;
          std::vector<NativeRemixPreset> deduped;
          deduped.reserve(seen.size());
          for (auto& [id, idx] : seen) deduped.push_back(std::move(remix[idx]));
          return deduped;
      }

      inline std::vector<NativeMashupPreset>
      load_mashup_presets()
      {
          std::vector<NativeRemixPreset>  tmp_remix;
          std::vector<NativeMashupPreset> mashup;
          load_builtins_from_binary(tmp_remix, mashup);
          load_user_dir_mashup(user_mashup_dir(), mashup);
          std::unordered_map<std::string, size_t> seen;
          for (size_t i = 0; i < mashup.size(); ++i) seen[mashup[i].id] = i;
          std::vector<NativeMashupPreset> deduped;
          deduped.reserve(seen.size());
          for (auto& [id, idx] : seen) deduped.push_back(std::move(mashup[idx]));
          return deduped;
      }

## Open Questions (revisit during APPLY)

- **Engine name field in remix JSON**: v3 has both `"engine": "chopped_screwed"`
  (engine-based) AND `"engine": "effect_chain"` (effect-chain). The
  parser keeps the string in NativeRemixPreset.engine — Phase 27
  decides which native engine to dispatch based on this.
- **Effect param schema**: v3 params dict is permissive. Our OpParams
  reads only known keys via `nlohmann::json::value(key, default)`. Extra
  keys are ignored silently. Should be sufficient for v3 preset
  compatibility.
- **BinaryData symbol mangling**: juce_add_binary_data converts each
  filename to a C identifier (e.g. `chopped_screwed_json`). The loader
  iterates `namedResourceList` so it doesn't hard-code these names.
- **User dir path**: `juce::File::userApplicationDataDirectory`
  resolves to `~/.config` on Linux, `~/Library/Application Support` on
