<objective>
## Goal
Enable CoreML EP on macOS when built with `-DAUTOREMIX_GPU=ON`, completing the GPU
acceleration story across all three platforms.

## Purpose
Apple Silicon M1/M2/M3 Macs represent a large share of DAW users. CoreML EP delegates
supported ONNX ops to the Neural Engine / GPU, targeting 3–10× CPU speedup — same
user-facing benefit as CUDA (Linux) and DirectML (Windows). The `onnxruntime-osx-universal2`
tarball already ships `coreml_provider_factory.h` and the CoreML EP provider; no new
dependency is needed.

## Output
- `CMakeLists.txt`: add `AUTOREMIX_COREML=1` compile definition for macOS GPU builds
- `NativeDemucsSeparator.h`: CoreML EP session path in `make_session` lambda +
  conditional `coreml_provider_factory.h` include
- `release.yml`: `build-macos-gpu` job producing `AutoRemix-macos-gpu-vX.zip`
- `README.md`: macOS row in GPU table updated from "CPU (coming v4.2)" to CoreML
- `CMakeLists.txt`: project version 4.1.0 → 4.2.0
</objective>

  <action>
    **CMakeLists.txt — macOS GPU block:**

    In the `elseif(APPLE)` block, the current code uses the same CPU tarball regardless
    of AUTOREMIX_GPU. Add a conditional definition for CoreML:

    ```cmake
    elseif(APPLE)
      set(ORT_URL "https://github.com/microsoft/onnxruntime/releases/download/v${ORT_VERSION}/onnxruntime-osx-universal2-${ORT_VERSION}.tgz")
      set(ORT_SHA "")
      set(ORT_LIB_SUBPATH "lib/libonnxruntime.${ORT_VERSION}.dylib")
      set(ORT_INCLUDE_SUBPATH "include")
      # CoreML EP ships in the universal2 tarball — no separate package needed.
      # AUTOREMIX_COREML signals NativeDemucsSeparator to attempt CoreML EP.
    endif()
    ```

    After the IMPORTED target setup block (after the `if(WIN32)` block at the end),
    add in `plugin/CMakeLists.txt` under the existing `AUTOREMIX_GPU` definition:

    Actually, add `AUTOREMIX_COREML=1` to the AutoRemix compile definitions in
    `plugin/CMakeLists.txt` using a generator expression:

    In `target_compile_definitions(AutoRemix PUBLIC ...)`, add:
    ```cmake
    $<$<AND:$<BOOL:${AUTOREMIX_GPU}>,$<PLATFORM_ID:Darwin>>:AUTOREMIX_COREML=1>
    ```

    This ensures AUTOREMIX_COREML is only defined on macOS when AUTOREMIX_GPU=ON.

    **NativeDemucsSeparator.h — CoreML include + session path:**

    After the existing DML include guard (lines ~34-36), add:
    ```cpp
    #if defined(AUTOREMIX_GPU) && defined(__APPLE__) && defined(AUTOREMIX_COREML)
    #include <coreml_provider_factory.h>
    #endif
    ```

    In the `make_session` lambda, inside the `#if defined(AUTOREMIX_GPU)` block,
    after the existing `#elif defined(__linux__)` CUDA block, add:
    ```cpp
    #elif defined(__APPLE__) && defined(AUTOREMIX_COREML)
    Ort::ThrowOnError(
        OrtSessionOptionsAppendExecutionProvider_CoreML(gpu_opts, 0));
    ```

    This completes the platform chain:
    - `_WIN32` → DirectML EP
    - `__linux__` → CUDA EP
    - `__APPLE__ && AUTOREMIX_COREML` → CoreML EP
    - (else: GPU flag set but no matching platform → falls through to CPU)

    Avoid: changing anything in the CPU EP path or the existing Linux/Windows blocks.
    Avoid: linking against CoreML framework directly — ORT loads it internally.
  </action>
