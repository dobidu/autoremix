# Phase 21-01 — AutoMashup Sidecar (MashupEngine + endpoint)

## Objective

Add a sidecar capability to combine two tracks into one coherent mashup:
separate both, analyze BPM + key, tempo-match and key-match, mux stems by
per-stem source selection, mix to a single output WAV.

Track A is the anchor (target BPM and target key default to A's analysis).
Track B is time-stretched + pitch-shifted to match.

This sub-plan covers Python only. No C++ work, no UI work.

## Acceptance Criteria

- [ ] `POST /api/v1/mashup` returns `{output_path, target_bpm, target_key, length_sec}`
- [ ] Request body validated by Pydantic model `MashupRequest`
- [ ] `stem_sources` accepts mapping `{vocals: "a"|"b", drums: ..., bass: ..., other: ...}`
- [ ] `target_bpm`, `target_key` optional in request — if absent, anchor to track A
- [ ] Both tracks separated using the same separator id (request field `separator_id`)
- [ ] Track B time-stretched by factor `target_bpm / bpm_b` (clamped 0.5–2.0)
- [ ] Track B pitch-shifted by semitone delta computed from key A vs key B (range −6..+6, wrap by ±12 to minimize)
- [ ] Output length = `min(length_a_after_stretch, length_b_after_stretch)`
- [ ] Stems muxed: for each stem name, output stem = stems_a[name] if source == "a" else stems_b_processed[name]
- [ ] Final mix = sum of 4 muxed stems, LUFS-normalized
- [ ] 6 pytest tests pass: schema validation, key delta calc (3 cases), stretch factor clamp, end-to-end happy path
- [ ] Long ML work wrapped in `asyncio.to_thread` (health endpoint must stay responsive)

## Tasks

### T1 — Pydantic models
- `MashupRequest` in `server/models.py`:
  - `file_a: str`, `file_b: str`
  - `separator_id: str = "algorithmic"`
  - `stem_sources: dict[str, Literal["a", "b"]]` (keys: vocals/drums/bass/other)
  - `target_bpm: float | None = None`
  - `target_key: str | None = None`
  - `output_dir: str | None = None`
- `MashupResponse`: `output_path: str`, `target_bpm: float`, `target_key: str`, `length_sec: float`

### T2 — Key-delta helper
- New function in `server/remix/analysis.py`: `semitone_delta(key_from: str, key_to: str) -> int`
- Parses `"C"`, `"C#"`, `"Am"`, etc. → root index 0–11 + minor flag
- Returns shortest semitone path in [-6, +6]
- Major/minor mismatch: ignore mode, shift root only (MVP — full Camelot wheel deferred)

### T3 — `MashupEngine`
- New file `server/remix/mashup.py` with `class MashupEngine`
- `process(file_a, file_b, separator, stem_sources, target_bpm, target_key, output_path)`:
  1. `stems_a = separator.separate(file_a, tempdir_a)`
  2. `stems_b = separator.separate(file_b, tempdir_b)`
  3. `bpm_a = detect_bpm(load(file_a))`, `bpm_b = detect_bpm(load(file_b))`
  4. `key_a = detect_key(...)`, `key_b = detect_key(...)`
  5. `target_bpm = target_bpm or bpm_a`
  6. `target_key = target_key or key_a`
  7. For each stem in stems_b: `time_stretch(factor=target_bpm/bpm_b)` then `pitch_shift(semitones=semitone_delta(key_b, target_key))`
  8. Per stem name, choose A or B based on `stem_sources[name]`
  9. Trim all 4 chosen stems to `min(len)`
  10. Sum stems, LUFS-normalize via `normalize_lufs` (existing helper in `base.py`)
  11. Write WAV via `soundfile.write` to `output_path`
- Returns `{target_bpm, target_key, length_sec}`

### T4 — Endpoint wiring
- In `server/main.py`: `@app.post("/api/v1/mashup")` handler
- Resolve separator via `SeparatorRegistry.get(separator_id)` — error 400 if not available
- Wrap engine call in `await asyncio.to_thread(...)` (no event-loop blocking)
- Return `MashupResponse`

### T5 — Tests
- `sidecar/tests/test_mashup.py`:
  - `test_mashup_request_schema_valid` — full body validates
  - `test_mashup_request_missing_stem_source_rejected` — Pydantic raises
  - `test_semitone_delta_same_key` — `delta("C", "C") == 0`
  - `test_semitone_delta_up_minor_third` — `delta("C", "Eb") == 3`
  - `test_semitone_delta_wraps_negative` — `delta("C", "A") == -3` (not +9)
  - `test_mashup_end_to_end` — fixture: two 5-sec sine WAVs at different BPMs; assert output exists, length matches min, key analysis shows shift applied (use `pytest.importorskip("librosa")`)

## Non-Goals

- No C++ / UI work — that is Phase 21-02 onward
- No streaming progress (mashup is synchronous in this sub-plan)
- No multi-track (>2) mashup
- No manual beat-offset slider
- No Camelot-wheel-aware mode-preserving key matching (mode-flip ignored for MVP)
- No loop/crossfade for length mismatch (truncate-to-shorter only)

## Open Questions (revisit during APPLY)

- Stretch factor clamp range: 0.5–2.0 OK, or tighter?
- Pitch-shift fallback when key detection low-confidence (single dominant chord) — accept noisy delta or detect "unknown" and skip shift?
- Separator caching: if same file_a appears in two consecutive mashup calls, should results be cached? (defer caching to a future phase; recompute for now)
