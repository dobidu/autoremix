<objective>
## Goal
Make the editor UI fully data-driven: StyleTabBar populated from `getPresets()` at startup; slider defaults and remix engine selection driven by `PresetInfo` data instead of hardcoded static arrays.

## Purpose
09-03 added `AudioBridge::getPresets()` but the UI still ignores it — three static arrays in the editor still hardcode labels, defaults, and engine IDs. This plan removes all three arrays and replaces them with live preset data, completing the preset wiring for Phase 09.

## Output
- `plugin/include/StyleTabBar.h` — dynamic label support (vector<string> instead of const array<char*,3>)
- `plugin/include/PluginEditor.h` — `presets_` member, `#include <vector>`
- `plugin/src/AutoRemixEditor.cpp` — bg-thread preset fetch at startup; `loadEngineDefaults` and `onClick_Play` driven by `presets_`
</objective>

<task type="auto">
  <name>Task 2: Wire editor to getPresets() — remove all 3 hardcoded arrays</name>
  <files>plugin/include/PluginEditor.h, plugin/src/AutoRemixEditor.cpp</files>
  <action>
    **PluginEditor.h:**

    Add `#include <vector>` after existing includes.
    Add member after `juce::String output_path_`:
    ```cpp
    std::vector<autoremix::PresetInfo> presets_;
    ```

    **AutoRemixEditor.cpp:**

    1. **Constructor** — after `drawAndConfigComponents()`, fetch presets on bg thread:
    ```cpp
    std::thread([this]() {
        auto presets = audioProcessor.getBridge().getPresets();
        if (presets.empty()) return;
        juce::MessageManager::callAsync([this, presets = std::move(presets)]() mutable {
            presets_ = std::move(presets);
            std::vector<std::string> names;
            names.reserve(presets_.size());
            for (auto& p : presets_) names.push_back(p.name);
            style_tab_.setLabels(std::move(names));
            loadEngineDefaults(0);
        });
    }).detach();
    ```

    2. **loadEngineDefaults** — replace static array entirely:
    ```cpp
    void AutoRemixAudioProcessorEditor::loadEngineDefaults(int idx) {
        if (presets_.empty() || idx < 0 || (size_t)idx >= presets_.size()) return;
        auto& p = presets_[(size_t)idx].default_params;
        tempo_slider_.setValue(p.tempo_factor,     juce::dontSendNotification);
        pitch_slider_.setValue(p.pitch_shift_semi, juce::dontSendNotification);
        reverb_slider_.setValue(p.reverb_mix,      juce::dontSendNotification);
        chop_slider_.setValue(p.chop_interval_ms,  juce::dontSendNotification);
    }
    ```

    3. **onClick_Play** — remove both static arrays (`engine_extras`, `engine_ids`); guard on empty presets:

    Replace the block that builds `params` and `output_path`:
    ```cpp
    auto idx = static_cast<std::size_t>(style_tab_.getSelectedIndex());
    if (presets_.empty() || idx >= presets_.size()) {
        status_lbl.setText("Presets not loaded — start sidecar first.", juce::dontSendNotification);
        return;
    }
    auto& preset = presets_[idx];
    autoremix::RemixParams params {
        (float)tempo_slider_.getValue(),
        (float)pitch_slider_.getValue(),
        (float)reverb_slider_.getValue(),
        (float)chop_slider_.getValue(),
        preset.default_params.bass_boost_db,
        preset.default_params.drums_tempo_factor,
        preset.id,
        "algorithmic"
    };

    auto input_file = juce::File(file_path_);
    std::filesystem::path output_path =
        std::filesystem::path("/tmp/autoremix/remix") /
        (input_file.getFileNameWithoutExtension().toStdString() + "_" + preset.id + ".wav");
    ```

    Remove `#include <array>` from AutoRemixEditor.cpp (no longer used).
    Keep `#include <thread>`, `#include <filesystem>`.

    Note: `presets_` is accessed only on the message thread (main thread). The bg thread in the constructor captures by value and only touches `presets_` inside `callAsync`. No mutex needed.
    Note: Consistent with existing `onClick_Play` pattern — bg thread captures `this` without atomic guard. 2s timeout makes this safe in practice.
