<objective>

## Goal
Make `ScreenSeparating` mashup-aware so the second track's separation
flows through the same 4-row + elapsed-timer progress UI as the first.
Removes the "frozen footer status" experience users hit when picking
track B with the Demucs separator.

## Purpose
Track A separation has a polished progress screen (4 stem rows, timer,
cancel). Track B currently only updates the footer status bar while the
StemsReady screen for track A keeps showing — feels frozen, especially
with Demucs taking 1–5 min on CPU.

## Output
- One new ctx_ flag: `mashup_mode_separating`
- ScreenSeparating reads the flag at onEnter and adapts:
  source file, target stems field, navigation target, header text
- Editor's `startMashupFlow()` rewritten: file picker → quick analyze
  (status bar) → set ctx_.file_path_b + flag → navigate to Separating
  (instead of running separation inline)
</objective>

## Open Questions (revisit during APPLY)

- **Should startMashupFlow keep the analyzeFile step before navigating?**
  Default: yes — it's fast (~1s for 60s of audio) and lets ScreenMashup
  show BPM + key immediately. Without it, ScreenMashup would show "BPM 120
  KEY --" until... never, since we'd skip analyze entirely.
- **Cancel destination from mashup separation**: StemsReady (track A's
  view, A is still loaded) vs Empty. Going to Empty would lose A's
  separation work — bad. Going to StemsReady is the right call.
