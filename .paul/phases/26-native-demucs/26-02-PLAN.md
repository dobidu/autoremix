<objective>

## Goal
Wire ONNX Runtime (ORT) C++ into the plugin build and add two
header-only modules:
- `ModelDownloader` — first-launch GET of `htdemucs.onnx` from the
  pinned release URL into the user cache dir, with SHA256 verification
  and retry-on-failure.
- `NativeDemucsSeparator` — loads the cached ONNX via `Ort::Session`,
  chunks input audio into the model's fixed 7.8 s window with 25 %
  overlap, runs ORT per chunk, and stitches outputs into the existing
  `separators::NativeStems { vocals, drums, bass, other }` struct.

## Purpose
This closes Phase 26. Phase 27 (the next phase) wires the native
separator into the screen flow, removes AudioBridge, and ships v4.0.0.
26-02's job is to make the native ML-separation path compile, download,
load, run, and produce the right tensor shape — without yet replacing
any v3 call site.

## Output
- CMake top-level: `FetchContent` per-platform ORT pre-built binary
  (Linux x86_64 / macOS universal2 / Win x64), imported as
  `onnxruntime::onnxruntime`. Default version pin: ORT 1.17.0.
- `plugin/include/dsp/ModelDownloader.h` — header-only,
  namespace `autoremix::dsp::models`. Public API:
    ```cpp
    struct DownloadStatus { bool ok; juce::String error; juce::File path; };
    DownloadStatus ensure_htdemucs(std::function<void(double)> progress = {});
    juce::File htdemucs_cache_path();
    ```
  Uses `juce::URL::createInputStream` + `juce::SHA256` + retry × 3.
- `plugin/include/dsp/NativeDemucsSeparator.h` — header-only,
  namespace `autoremix::dsp::separators`. Public API:
    ```cpp
    struct DemucsResult { bool ok; juce::String error; NativeStems stems; };
    DemucsResult separate_demucs(const juce::AudioBuffer<float>& input,
                                 double                          sample_rate,
                                 const juce::File&               model_path,
                                 std::function<void(double)>     progress = {});
    ```
  Internals: chunk loop (7.8 s window, 25 % overlap, raised-cosine
  cross-fade), `Ort::Session` reused across chunks, fixed input shape
  `[1, 2, 343980]` per chunk.
- Both new headers smoke-included from `AutoRemixProcessor.cpp` —
  no call sites this plan (those land in 27-01).

  <files>plugin/include/dsp/NativeDemucsSeparator.h, plugin/src/AutoRemixProcessor.cpp</files>
  <action>
    Create `plugin/include/dsp/NativeDemucsSeparator.h`. Header-only,
    namespace `autoremix::dsp::separators`. Public API:

      struct DemucsResult {
          bool         ok = false;
          juce::String error;
          NativeStems  stems;
      };

      inline DemucsResult separate_demucs(
          const juce::AudioBuffer<float>& input,
          double                          sample_rate,
          const juce::File&               model_path,
          std::function<void(double)>     progress = {});

    Implementation outline (don't ship vague pseudocode — write the
    real code, including ORT session setup, chunk loop, raised-cosine
    crossfade, and final stem repacking):

      Constants:
          static constexpr int   kModelWindowSamples = 343'980; // 7.8 s @ 44.1 kHz
          static constexpr int   kModelSampleRate    = 44'100;
          static constexpr float kOverlapFraction    = 0.25f;
          static constexpr int   kSourcesPerStem     = 4;
          static constexpr int   kModelChannels      = 2;

      Steps inside separate_demucs:
          1. Pre-flight checks: model_path exists; sample_rate == 44100
             (else: return error — the v3 sidecar resampled with
             torchaudio; we punt on resampling for first cut and
             require 44.1 kHz callers). Document this clearly.
          2. Convert input AudioBuffer<float> to [1, 2, N] float32
             column-major contiguous (mono → duplicate channel).
          3. Build Ort::Env once, Ort::SessionOptions with
             `SetGraphOptimizationLevel(ORT_ENABLE_ALL)`, `SetIntraOpNumThreads(0)`,
             and an `Ort::Session(env, model_path.getFullPathName().toRawUTF8(), opts)`.
          4. Allocate output stems: 4 × AudioBuffer<float>(2, N), zeroed.
          5. Compute chunk schedule:
                hop = kModelWindowSamples * (1 - kOverlapFraction);  // 257985
                num_chunks = ceil((N - kModelWindowSamples) / hop) + 1;
                (handle N < window by single zero-padded chunk)
          6. Pre-compute raised-cosine fade-in / fade-out windows
             (length = kModelWindowSamples * kOverlapFraction).
          7. Loop chunks:
                offset = chunk_idx * hop
                pad start/end if at boundaries
                build input tensor [1, 2, kModelWindowSamples]
                run session
                read output tensor [1, 4, 2, kModelWindowSamples]
                apply fade-in on the leading overlap, fade-out on the trailing overlap
                add into output stems at [offset .. offset+window]
                progress((chunk_idx + 1) / num_chunks)
          8. Trim output stems to input length N.
          9. Pack into NativeStems and return.
          10. Wrap the entire body in a try / catch (const Ort::Exception&)
              → return DemucsResult{false, e.what(), {}}.

      Include order:
          #include <onnxruntime_cxx_api.h>
          #include <juce_audio_basics/juce_audio_basics.h>
          #include "NativeAlgorithmicSeparator.h"  // NativeStems

      All ORT C++ types live in the `Ort::` namespace. No raw pointers
      escape the function.

    Add `#include "dsp/NativeDemucsSeparator.h"  // Phase 26-02 smoke`
    to `plugin/src/AutoRemixProcessor.cpp`. Do NOT call separate_demucs
    from any screen yet — that's 27-01.

## Open Questions (revisit during APPLY)

- **ORT SHA256 pinning**: leaving as a placeholder is fragile.
  FetchContent will refetch on every clean build and may break if
  Microsoft re-uploads the tarball. The checkpoint asks the user
  for the SHA after the first fetch; we then update the plan.
  Alternative: skip URL_HASH (deviation, log explicitly).

- **Resampling**: the v3 Python path used torchaudio.functional.resample
  for non-44.1 kHz inputs. C++ side options:
    a) Require 44.1 kHz input, return an error otherwise. Simplest.
    b) Use juce::Interpolators::WindowedSinc for resampling.
    c) Add a libsamplerate dependency.
  Recommend (a) for first cut — DAWs typically host at 44.1 / 48 kHz
  and the screens can resample upstream if needed. Note in the
  separator header.

- **ORT execution providers**: this plan does CPU only. CUDA / DirectML
  would require pulling the platform-specific extended-EP tarballs
  and significantly more CMake logic. Out of scope. Note in
  PROJECT.md as a possible v4.1 enhancement.

- **Manual smoke test**: AC-3 wants "produces the right tensor shape"
  but the cleanest proof is "stems sound like demucs output". Without
  wiring screens (which is 27-01), the easiest manual test is a
  ~50-line scratch program in tests/. Decision deferred to the
  human-verify checkpoint — if the user wants more confidence, we
  add the scratch test then.

- **Plugin binary size impact**: ORT libonnxruntime.so on Linux x64
  v1.17.0 is ~14 MB. The plugin .so + ORT .so + (downloaded) model
  ~= 360 MB on first use. Document in PROJECT.md after 26-02 lands.
