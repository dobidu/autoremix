<objective>
## Goal
Add in-plugin audio preview: after remixing completes, the user can press a "Preview"
button to play the remixed WAV through the plugin's audio output without exporting first.
Playback is driven by `AudioTransportSource` in the processor; the editor controls it via
public methods.

## Purpose
Currently users must save the file and open it externally to hear the result. Preview
closes the creative loop — hear the remix, tweak sliders, remix again — without leaving
the plugin. This is the core interactive mixing UX of Phase 13.

## Output
- `AutoRemixAudioProcessor` gains: `AudioFormatManager`, `AudioFormatReaderSource`,
  `AudioTransportSource`; four public methods; `prepareToPlay/releaseResources/processBlock`
  wired for transport playback.
- `AutoRemixAudioProcessorEditor` gains: "Preview" button; auto-loads output file after
  remix completes; "Play" button renamed "Remix" to reduce UX confusion.
</objective>

    ```cpp
    void AutoRemixAudioProcessor::loadPreviewFile(const juce::File& f)
    {
        transport_.stop();
        transport_.setSource(nullptr);
        reader_source_.reset();
        if (auto* reader = format_manager_.createReaderFor(f)) {
            reader_source_ = std::make_unique<juce::AudioFormatReaderSource>(reader, true);
            transport_.setSource(reader_source_.get(), 0, nullptr, reader->sampleRate);
        }
    }

    void AutoRemixAudioProcessor::togglePreview()
    {
        if (transport_.isPlaying())
            transport_.stop();
        else {
            transport_.setPosition(0.0);
            transport_.start();
        }
    }

    void AutoRemixAudioProcessor::stopPreview() { transport_.stop(); }
    bool AutoRemixAudioProcessor::isPreviewPlaying() const { return transport_.isPlaying(); }
    ```
