<objective>
## Goal
Add four stem-mix gain sliders (Vocals / Drums / Bass / Other, 0.0–2.0) to the plugin
UI. The selected weights are sent to the sidecar as `stem_mix_override` and applied as
a pre-weighting step before any engine or effect chain runs — universal across all
remix paths.

## Purpose
Stem separation delivers four independent tracks; users should control their balance.
Currently the preset's static `stem_mix` is the only lever and cannot be adjusted
without editing JSON. The sliders expose this in real time.

## Output
- `RemixRequest.stem_mix_override` optional dict in Python sidecar
- `_apply_stem_weights()` helper in `main.py` — reads each stem WAV, scales by factor, writes to temp weighted WAV, returns new StemPaths
- `RemixParams` in C++ gains four floats: `vocals_gain`, `drums_gain`, `bass_gain`, `other_gain` (all default 1.0f)
- `AudioBridge::applyRemix()` includes `stem_mix_override` in JSON body
- Controls zone reorganised into two columns: left = remix params, right = stem mix
- File label moved to its own dedicated row above both columns
</objective>

    def _apply_stem_weights(stems: StemPaths, weights: dict[str, float], out_dir: Path) -> StemPaths:
        """Scale each stem WAV by its weight factor; write to out_dir. Return new StemPaths."""
        import soundfile as sf
        import numpy as np
        out_dir.mkdir(parents=True, exist_ok=True)
        result = {}
        for name in ("vocals", "drums", "bass", "other"):
            path = getattr(stems, name)
            audio, sr = sf.read(str(path), dtype="float32", always_2d=True)
            factor = float(weights.get(name, 1.0))
            sf.write(str(out_dir / f"{name}.wav"), audio * factor, sr)
            result[name] = out_dir / f"{name}.wav"
        return StemPaths(
            vocals=result["vocals"], drums=result["drums"],
            bass=result["bass"],    other=result["other"],
        )
    ```

    In the `remix()` route, immediately after `stems = req.to_stems()` and before

    ```
