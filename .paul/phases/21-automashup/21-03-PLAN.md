<objective>

## Goal
Build the `ScreenMashup` UI: a single screen that shows track-A and track-B
analysis, lets the user pick a per-stem source (A or B), set target BPM and
target key, then triggers the sidecar mashup pipeline via `ctx_.run_mashup`
(wired in 21-02).

## Purpose
Surfaces the pairwise-mashup capability delivered in 21-01/02 to the user.
This screen is **reachable from nowhere yet** — 21-04 adds the entry point
(button on `ScreenStemsReady` + 2nd-file load flow). 21-03 ships the screen
in isolation so 21-04 only needs to wire navigation, not build UI.

## Output
- New `ScreenId::Mashup` enum value
- New `plugin/include/ScreenMashup.h` (header-only screen class)
- Editor `navigateTo()` switch case constructing `ScreenMashup` with run callback
- Build green on Linux (Shared + VST3 + Standalone) and macOS (+ AU)
</objective>

## AC-1: Screen renders track A + track B analysis

```gherkin
Given ScreenMashup is shown
  And ctx_.file_path (track A) is set with detected_bpm and detected_key
  And ctx_.file_path_b (track B) is set with detected_bpm_b and detected_key_b
When the screen paints
Then both tracks' filenames, BPM, and key are visible
  And there is a clear visual separation between the A and B blocks
```

## AC-2: Per-stem source picker reflects + writes ctx_.mashup_sources

```gherkin
Given ScreenMashup is shown
  And ctx_.mashup_sources is {vocals:"a", drums:"b", bass:"b", other:"b"}
When the screen first appears
Then for each stem, the "A" or "B" button matching the current source is highlighted
When the user clicks "A" on the drums row
Then ctx_.mashup_sources["drums"] becomes "a"
  And the drums "A" button is highlighted, "B" is not
```

## AC-3: Target BPM and target key controls write ctx_

```gherkin
Given ScreenMashup is shown
When the user moves the target BPM slider to 130
Then ctx_.mashup_target_bpm becomes 130.0
When the user selects "Anchor to A" in the target-key combo
Then ctx_.mashup_target_key becomes empty (signals: anchor)
When the user selects "F#" in the target-key combo
Then ctx_.mashup_target_key becomes "F#"
```

## AC-4: Generate button invokes run_mashup and navigates to Render

```gherkin
Given ScreenMashup is shown
  And ctx_.run_mashup is wired
When the user clicks "Generate Mashup"
Then the button disables
  And the status label shows "Mashing up..."
  And ctx_.run_mashup is called with a MashupParams whose:
    - file_a = ctx_.file_path
    - file_b = ctx_.file_path_b
    - stem_sources = ctx_.mashup_sources
    - has_target_bpm = true; target_bpm = ctx_.mashup_target_bpm
    - target_key     = ctx_.mashup_target_key (may be empty)
When the result returns successfully
Then ctx_.mashup_output_path is set
  And the screen navigates to ScreenId::Render (reusing the existing Done state)
When the result returns with success=false
Then the status label shows the error and the button re-enables
```

<task type="auto">
  <name>Task 2: Create ScreenMashup.h with widgets, layout, and submit logic</name>
  <files>plugin/include/ScreenMashup.h</files>
  <action>
    Header-only class `ScreenMashup : public ScreenBase`. Follow the
    `ScreenModeParams` pattern.

    Constructor signature:
      `ScreenMashup(ScreenContext& ctx)`
      (the run_mashup callback is read from `ctx_.run_mashup` at click time)

    Widgets (all juce::TextButton / juce::Slider / juce::ComboBox / juce::Label):
    - `header_lbl_` "MASHUP" (top, section-label font)
    - Track A block (left half):
        `a_filename_lbl_`, `a_bpm_lbl_`, `a_key_lbl_`
    - Track B block (right half):
        `b_filename_lbl_`, `b_bpm_lbl_`, `b_key_lbl_`
    - 4 stem-source rows (vocals/drums/bass/other), each with:
        `<name>_lbl_`, `<name>_a_btn_`, `<name>_b_btn_`
        Clicking `a_btn_` sets ctx_.mashup_sources[name]="a", styles A as active
        Clicking `b_btn_` sets ctx_.mashup_sources[name]="b", styles B as active
        Use setComponentID("ghost") on the inactive one, default look on the active one
        (matches existing button-pair styling pattern)
    - Target BPM:
        `target_bpm_lbl_` "TARGET BPM", `target_bpm_slider_` range 40–200
        onValueChange → ctx_.mashup_target_bpm
    - Target Key:
        `target_key_lbl_` "TARGET KEY", `target_key_combo_`
        Items: "Anchor to A", "C", "C#", "D", "Eb", "E", "F", "F#", "G", "Ab", "A", "Bb", "B"
        Selecting "Anchor to A" → ctx_.mashup_target_key = ""
        Otherwise → ctx_.mashup_target_key = selected note string
    - `back_btn_` "< Back" (returns to ScreenId::StemsReady — track A's stem screen)
    - `generate_btn_` "Generate Mashup >" (primary action)
    - `status_lbl_` (bottom, mono, hidden initially)

    onEnter():
      - Populate A labels from ctx_.file_path / detected_bpm / detected_key
      - Populate B labels from ctx_.file_path_b / detected_bpm_b / detected_key_b
      - Set target_bpm_slider_ to ctx_.detected_bpm (default anchor to A)
      - Apply ctx_.mashup_sources to button-pair styling (4 rows)
      - Set target_key_combo_ index 0 ("Anchor to A") if ctx_.mashup_target_key empty

    handleGenerate():
      - generate_btn_.setEnabled(false)
      - status_lbl_.setVisible(true); status_lbl_.setText("Mashing up...", dontSendNotification)
      - Build autoremix::MashupParams from ctx_ fields:
          file_a = std::filesystem::path(ctx_.file_path.toStdString())
          file_b = std::filesystem::path(ctx_.file_path_b.toStdString())
          separator_id = ctx_.separators[ctx_.selected_separator_idx].id (fallback "algorithmic")
          stem_sources = ctx_.mashup_sources
          target_bpm = ctx_.mashup_target_bpm; has_target_bpm = true
          target_key = ctx_.mashup_target_key.toStdString()
      - Call ctx_.run_mashup(params, [this](MashupResult r) { handleResult(r); })

    handleResult(MashupResult):
      - On success: ctx_.mashup_output_path = juce::String(r.output_path.string());
                    ctx_.output_path = ctx_.mashup_output_path;  // so ScreenRender Done picks it up
                    if (ctx_.navigate) ctx_.navigate(ScreenId::Render);
      - On failure: status_lbl_.setText("Error: " + r.error_message, dontSendNotification)
                    generate_btn_.setEnabled(true)

    paint(): background fill (AR::BG). Section dividers as 1px lines between A and B blocks.

    resized(): single-column top-down layout, 960×520. Top header ~40px, A/B blocks ~80px,
               stem rows ~36px × 4 = 144px, target BPM ~40px, target key ~40px,
               status ~24px, footer buttons ~48px.

    Use AR::COMMENT / AR::FG / AR::ACCENT color tokens from AutoRemixLookAndFeel.h.
    Use AR::font(AR::FontRole::section_label) for section labels.
    Wrap everything in `JUCE_DECLARE_NON_COPYABLE_WITH_LEAK_DETECTOR(ScreenMashup)`.
  </action>
