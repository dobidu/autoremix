<objective>
## Goal
Replace the raw `debug_text` TextEditor and nonfunctional `stop_btn` with a clean
`status_lbl` (one-line user-facing message) and an indeterminate `juce::ProgressBar`
that spins during processing and hides when idle.

## Purpose
The current UI exposes internal debug strings ("Separating...\n", "Error: ...") in a
raw multi-line editor — wrong for a production plugin. Users need a single clean status
line and a visual indicator that work is happening.

## Output
- `plugin/include/PluginEditor.h` — `debug_text` + `stop_btn` removed; `status_lbl`,
  `progress_bar_`, `progress_` added
- `plugin/src/AutoRemixEditor.cpp` — layout updated, progress driven from `onClick_Play`
  callAsync calls, Stop removed
</objective>
