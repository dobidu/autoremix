## Goal
Define the Effect Chain DSL: update `schema.json` with a typed `effects` array spec, and write `design-spec.md` documenting available operations, execution model, and worked examples — making 10-02 (interpreter) executable without design ambiguity.

## Purpose
The `effects: []` forward-compat hook has existed since Phase 09. This plan gives it a concrete, implementable shape. Without a written spec, 10-02 would require guessing op names, parameter shapes, and execution semantics at implementation time — producing an interpreter that may not match what preset authors expect.

- `.paul/phases/10-effect-chain/design-spec.md` — authoritative spec: op catalogue, execution model, backward-compat rules, two worked examples
</objective>

<task type="auto">
  <name>Task 2: Write design-spec.md — op catalogue + execution model</name>
  <files>.paul/phases/10-effect-chain/design-spec.md</files>
  <action>
    Create `.paul/phases/10-effect-chain/design-spec.md` with these sections:

    **1. Overview** — DSL purpose, relation to engine-based dispatch, version compatibility.

    **2. Execution Model**
    - When `effects` array is non-empty: engine field ignored, interpreter runs chain
    - Stems loaded as separate numpy arrays (shape: [2, N] stereo)
    - Each op receives current stem buffer(s), returns modified buffer(s)
    - Op targeting `"all"`: applies to the summed mix of all stems (mix-then-process)
    - Op targeting a single stem or list: applies only to those stem buffers
    - After chain completes: `stem_mix` weights applied (element-wise multiply), then stems summed and written
    - When `effects` is empty: fall through to `engine` field dispatch (Phase 09 behaviour)

    **3. Op Catalogue** — for each of the 6 ops:

    | Op | params | stems | librosa/pedalboard call |
    |----|--------|-------|------------------------|
    | `time_stretch` | `factor: float [0.1–4.0]` | any | `librosa.effects.time_stretch(audio, rate=factor)` |
    | `pitch_shift` | `semitones: float [-24–24]` | any | `librosa.effects.pitch_shift(audio, sr, n_steps=semitones)` |
    | `reverb` | `mix: float [0–1]`, `room_size: float [0–1] default 0.6` | any | `pedalboard.Reverb(room_size, wet_level=mix)` |
    | `chop` | `interval_ms: float [100–10000]` | any | existing `_chop()` logic from ChoppedAndScrewedEngine |
    | `bass_boost` | `db: float [-24–24]` | any (typically bass/drums) | `pedalboard.LowShelfFilter(cutoff_hz=200, gain_db=db)` |
    | `eq_highpass` | `cutoff_hz: float [20–20000]` | any (typically other/vocals) | `pedalboard.HighpassFilter(cutoff_frequency_hz=cutoff_hz)` |

    For each op, show a JSON snippet example.

    **4. Backward Compatibility**
    - Presets with `effects: []` (empty) are processed via `engine` as before — no change
    - Presets omitting `effects` key also use engine dispatch
    - Mixing engine + non-empty effects in one preset: effects take precedence (engine ignored)

    **5. Worked Examples**

    Example A — Custom Chopped & Screwed (equivalent to built-in engine):
    ```json
    {
      "id": "my_chop",
      "version": "2.0",
      "name": "My Chop",
      "params": { "tempo_factor": 0.70, ... },
      "effects": [
        {"op": "time_stretch", "stems": "all", "params": {"factor": 0.70}},
        {"op": "pitch_shift",  "stems": "all", "params": {"semitones": -4.0}},
        {"op": "chop",         "stems": "all", "params": {"interval_ms": 2000}},
        {"op": "reverb",       "stems": "all", "params": {"mix": 0.05}}
      ]
    }
    ```

    Example B — Stems-targeted DnB:
    ```json
    {
      "id": "my_dnb",
      "version": "2.0",
      "name": "My DnB",
      "params": { "tempo_factor": 1.4, ... },
      "effects": [
        {"op": "time_stretch", "stems": ["vocals", "other"], "params": {"factor": 1.4}},
        {"op": "time_stretch", "stems": ["drums"],           "params": {"factor": 2.0}},
        {"op": "bass_boost",   "stems": ["bass", "drums"],   "params": {"db": 6.0}},
        {"op": "eq_highpass",  "stems": ["other"],           "params": {"cutoff_hz": 300}}
      ]
    }
    ```

    **6. Implementation Notes for 10-02**
    - New file: `python/server/remix/chain_interpreter.py` — `EffectChainEngine` class
    - `EffectChainEngine.process(stems, preset, output_path)` — loads stems as numpy, runs chain, writes output
    - Registry: detect at remix time if preset has non-empty effects → route to interpreter instead of engine lookup
    - Error handling: unknown op → raise ValueError with op name; missing required param → raise ValueError
  </action>
