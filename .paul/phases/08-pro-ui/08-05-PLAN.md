<objective>
## Goal
Embed Inter font into the plugin binary via `juce_add_binary_data` and load it in `AutoRemixLookAndFeel` by overriding `getTypefaceForFont`. Replace system sans-serif with Inter across all AR::font() roles.

## Purpose
System font varies per host OS/DAW. Inter embedded in the binary gives identical rendering everywhere and matches the Dracula-inspired aesthetic better than whatever the host happens to provide.

## Output
- `assets/fonts/` — 3 OTF files copied from system
- `plugin/CMakeLists.txt` — `juce_add_binary_data(AutoRemixFonts ...)` + link
- `plugin/include/AutoRemixLookAndFeel.h` — typeface members + `getTypefaceForFont` override; `AR::font()` updated to produce fonts with the embedded typeface
</objective>

<task type="auto">
  <name>Task 3: Load Inter in AutoRemixLookAndFeel</name>
  <files>plugin/include/AutoRemixLookAndFeel.h</files>
  <action>
    Add `#include <BinaryData.h>` after the existing includes.

    Add typeface members to `AutoRemixLookAndFeel`:
    ```cpp
    juce::Typeface::Ptr inter_regular_;
    juce::Typeface::Ptr inter_semibold_;
    ```

    In constructor, after existing `setColour(...)` calls, load typefaces:
    ```cpp
    inter_regular_  = juce::Typeface::createSystemTypefaceFor(
        BinaryData::Inter_Regular_otf,  BinaryData::Inter_Regular_otfSize);
    inter_semibold_ = juce::Typeface::createSystemTypefaceFor(
        BinaryData::Inter_SemiBold_otf, BinaryData::Inter_SemiBold_otfSize);
    ```

    Add `getTypefaceForFont` override — bold roles (header/section/button use `juce::Font::bold`)
    get SemiBold; plain roles get Regular:
    ```cpp
    juce::Typeface::Ptr getTypefaceForFont(const juce::Font& f) override {
        return (f.getStyleFlags() & juce::Font::bold) ? inter_semibold_ : inter_regular_;
    }
    ```

    Update `AR::font()` to strip the bold style flag (we control weight via typeface, not style flags,
    so the flag only needs to guide typeface selection — but JUCE may apply synthetic bold on top if the
    flag is set and the typeface is already bold-weight). Safest: keep flags as-is (they guide
    `getTypefaceForFont`) but add `withStyle(0)` after typeface selection is done — actually this is not
    needed since `getTypefaceForFont` runs first. Leave `AR::font()` unchanged; the override handles it.
  </action>
  <verify>cmake --build build --parallel 2>&1 | tail -6</verify>
  <done>AC-1, AC-2, AC-3: Inter loaded, getTypefaceForFont routes by weight, build clean</done>
</task>
