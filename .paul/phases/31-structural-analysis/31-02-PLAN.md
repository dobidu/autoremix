<objective>
## Goal
Complete `analyze_structure()` by filling the three remaining fields:
`brightness_per_bar` (spectral centroid), `vocal_presence_per_bar` (1-4 kHz
energy heuristic), and `section_boundaries` (spectral novelty peaks).

## Purpose
These three fields are what distinguish a structural analysis from a simple
beat tracker. `brightness_per_bar` tracks timbral tension. `vocal_presence_per_bar`
lets templates place samples and scratches in instrumental gaps. `section_boundaries`
enables the template engine to map arrangement slots to real structural transitions
rather than guessing at fixed bar counts.

## Output
- `NativeAnalysis.h`: `spectral_centroid_per_bar()` + `vocal_presence_per_bar()` +
  `section_boundaries()` via spectral novelty. `analyze_structure()` updated to
  fill all remaining fields.

  <action>
    Add two helper functions to the `autoremix::dsp::analysis` namespace.

    **`spectral_centroid_per_bar()`**:
    ```cpp
    inline std::vector<float>
    spectral_centroid_per_bar(const juce::AudioBuffer<float>& audio, double sr,
                              const std::vector<double>& bar_times)
    ```
    Algorithm:
    - Convert to mono
    - For each bar window [bar_times[i], bar_times[i+1]):
      - Compute STFT using stft_magnitudes() over that window only
        (pass the mono sub-slice directly; use kFftSize/kHop constants)
      - Compute spectral centroid per frame: sum(bin * mag[bin]) / sum(mag[bin])
        where bin frequency = bin * sr / kFftSize
      - Average centroids across frames in the bar
    - Normalize all bar centroids to [0, 1] (divide by max centroid)
    - Return vector<float>

    STFT on sub-slices: create a temporary vector<float> for the bar window,
    call stft_magnitudes(). If bar is shorter than kFftSize, return 0.0 for that bar.

    **`vocal_presence_per_bar()`**:
    ```cpp
    inline std::vector<float>
    vocal_presence_per_bar(const juce::AudioBuffer<float>& audio, double sr,
                           const std::vector<double>& bar_times)
    ```
    Algorithm:
    - Heuristic: vocal energy = RMS energy in 1000–4000 Hz band vs. total RMS
    - For each bar window:
      - Extract mono samples for the bar
      - Compute band energy in [1000, 4000] Hz using a 2nd-order IIR bandpass
        (use juce::dsp::IIR::Coefficients<float>::makeBandPass(sr, 2500.0f, 0.7f))
        Apply via apply_iir_filter() (already in detail namespace)
      - Compute RMS of bandpass output / RMS of full signal
      - Clamp result to [0, 1]
    - Normalize across all bars to [0, 1]
    - Return vector<float>

    For short bars (< 512 samples): return 0.5 (neutral, avoid division-by-zero).
    Avoid: running full-song STFT for each bar — use windowed approach per bar.
  </action>

  <action>
    **`section_boundaries()`** — spectral novelty peak picking:
    ```cpp
    inline std::vector<int>
    section_boundaries(const std::vector<float>& energy_per_bar,
                       const std::vector<float>& brightness_per_bar,
                       int min_sep_bars = 4)
    ```
    Algorithm (simplified novelty — no full self-similarity matrix needed):
    - Novelty signal = weighted sum of frame-to-frame differences in energy + brightness:
      ```
      novelty[i] = |energy_per_bar[i] - energy_per_bar[i-1]|
                 + |brightness_per_bar[i] - brightness_per_bar[i-1]|
      ```
      (where i starts at 1; novelty[0] = 0)
    - Smooth novelty with a 3-point moving average to reduce jitter
    - Peak pick on smoothed novelty with min_sep_bars separation
      (same logic as energy_peaks: local max + min distance)
    - Threshold: only keep peaks above mean(novelty) * 1.2
    - Return indices (these are bar indices, NOT sample positions)

    Note: this simplified novelty avoids a full self-similarity matrix O(N²).
    It captures energy and brightness transitions well enough for template routing.
    A full self-similarity approach can be added later if needed.

    **Complete `analyze_structure()`** — add the three new fields:
    In the existing `analyze_structure()` function body, after the energy_peaks line,
    add:
    ```cpp
    s.brightness_per_bar      = spectral_centroid_per_bar(audio, sr, s.bar_times);
    s.vocal_presence_per_bar  = vocal_presence_per_bar(audio, sr, s.bar_times);
    s.section_boundaries      = section_boundaries(s.energy_per_bar,
                                                   s.brightness_per_bar, 4);
    ```

    Avoid: computing STFT twice at the top level — spectral_centroid_per_bar uses
    its own windowed STFT per bar. This is intentional (per-bar resolution vs.
    global resolution). Accept the cost.
