<objective>
## Goal
Add a Cancel button to ScreenSeparating that interrupts an in-progress Demucs
ML separation between chunks, returning control to ScreenEmpty immediately.

## Purpose
Demucs separation takes 30–120 s on CPU. Users cannot currently abort it — the UI
is locked until completion or error. A cancel mechanism is standard UX for long
background operations and prevents users from force-quitting the plugin.

## Output
- `NativeDemucsSeparator.h`: `separate_demucs` accepts optional `const std::atomic<bool>*`
  cancel token; checks between chunks; returns `DemucsResult{cancelled=true}` on cancel
- `ScreenContext.h`: `separation_cancel_token` shared_ptr for inter-component signalling
- `ScreenSeparating.h`: Cancel button; initializes token; navigates to Empty on cancel
- `AutoRemixEditor.cpp`: `separateNative` reads `ctx_.separation_cancel_token` for demucs path
</objective>

    for (int idx = 0; idx < num_chunks; ++idx) {

            result.error = "Cancelled";
            return result;

        // ... existing chunk processing ...

    is done on a cancelled iteration.
