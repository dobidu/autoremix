<objective>

## Goal
Add mashup templates: a JSON-based preset format for mashups (mirroring the
existing remix preset system) plus four new "feel knobs" applied in the
sidecar pipeline. Ship 8 built-in templates. Surface them in `ScreenMashup`
via a template combo + Advanced ▾ disclosure for the new knobs.

## Purpose
Turn the mashup screen from a blank-canvas mixer into an instantly-creative
tool. One click on "Vocal Acapella", "Slowed Mashup", or "Nightcore Mashup"
pre-fills all 8 gains + target BPM/key behavior + reverb + pitch shift
parameters. Power users can still customize via the Advanced section or
write their own JSON files in `~/.config/autoremix/mashup/`.

## Output
- `MashupPreset` Pydantic model + 4 new optional `MashupRequest` fields
- `MashupPresetLoader` (mirrors `PresetLoader`)
- 8 built-in template JSON files in `sidecar/server/mashup_presets/`
- `GET /api/v1/mashup_presets` endpoint
- `MashupPresetInfo` C++ struct + `AudioBridge::getMashupPresets()`
- ScreenMashup combo at top + Advanced ▾ disclosure with 5 extra sliders
- 4 new pytest cases for preset loading + feel-knob processing
</objective>

## Open Questions (revisit during APPLY)

- **Built-in template count**: 8 listed. Drop any? Add any? Currently a
  vocal-acapella, drum-swap, slowed, nightcore, dub-echo, instrumental-layer,
  bass-swap, frankenstein mix.
- **Master pitch + reverb application order**: pitch first, then reverb.
  Reverse if reverb-then-pitch feels better — easy swap in MashupEngine.
- **HPF range default**: 0–400 Hz. Above 400 starts removing mids; below 40
  is barely audible. Reasonable bracket.
- **Tempo modifier range**: 0.5–1.5. Tighter than the engine's hard clamp
  (0.5–2.0) but musically more useful — going past ±50% on the master starts
  to sound broken.
