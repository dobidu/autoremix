<objective>
## Goal
Add GPU execution provider to NativeDemucsSeparator: CUDA EP on Linux, DirectML EP on
Windows. Controlled by `-DAUTOREMIX_GPU=ON` CMake flag. CPU EP fallback when GPU EP
session construction fails at runtime.

## Purpose
Demucs CPU inference takes 30–120 s on typical hardware. GPU targets <10 s on a
mid-range GPU. This is the headline v4.1 user-facing improvement. The CPU path
(AUTOREMIX_GPU=OFF, the default) stays byte-for-byte identical.

## Output
- `CMakeLists.txt`: conditional GPU ORT package FetchContent + IMPORTED targets
- `plugin/include/dsp/NativeDemucsSeparator.h`: `make_session()` helper with GPU EP
  attempt → CPU EP fallback; conditional `dml_provider_factory.h` include
</objective>

<task type="checkpoint:decision" gate="blocking">
  <decision>Include macOS CoreML EP in Phase 28, or defer to Phase 29?</decision>
  <context>
    CoreML EP is already bundled in the standard onnxruntime-osx-universal2 package —
    no extra FetchContent needed. Enabling it requires one SessionOptions flag plus
    `#include <coreml_provider_factory.h>`. Apple Silicon gets Neural Engine acceleration
    (~3–5× faster than CPU). macOS is a large segment of DAW users.

    However: CoreML EP for ONNX models with custom ops (our export uses patched ops) has
    known compatibility gaps in ORT 1.17. May require fallback investigation. Adds
    Obj-C++ compile unit if we need to query CoreML availability. Risk of scope slip.
  </context>
  <options>
    <option id="include-coreml">
      <name>Include CoreML in Phase 28</name>
      <pros>Complete GPU story across all 3 platforms. Apple Silicon speedup ships with v4.1.</pros>
      <cons>Wider scope; CoreML EP + our patched ONNX export may need compatibility investigation before shipping.</cons>
    </option>
    <option id="defer-coreml">
      <name>Defer CoreML to Phase 29</name>
      <pros>Tight scope. Linux + Windows cover most production studios. Ship faster.</pros>
      <cons>macOS users miss GPU speedup until Phase 29.</cons>
    </option>
  </options>
  <resume-signal>Select: include-coreml or defer-coreml</resume-signal>

<task type="auto">
  <name>T3: NativeDemucsSeparator — GPU EP selection + CPU fallback</name>
  <files>plugin/include/dsp/NativeDemucsSeparator.h</files>
  <action>
    Add conditional include after existing ORT include (line 32):
    ```cpp
    #if defined(AUTOREMIX_GPU) && defined(_WIN32)
    #include <dml_provider_factory.h>
    #endif
    #if defined(AUTOREMIX_GPU) && defined(__APPLE__) && defined(AUTOREMIX_COREML)
    #include <coreml_provider_factory.h>
    #endif
    ```

    In separate_demucs(), replace the existing Ort::Env + Ort::SessionOptions + Ort::Session
    block (currently lines ~187–200: env creation through session construction) with:

    ```cpp
    Ort::Env env(ORT_LOGGING_LEVEL_WARNING, "autoremix-demucs");

    // Returns a session with GPU EP if available, CPU EP as fallback.
    auto make_session = [&](bool try_gpu) -> Ort::Session {
    #if defined(AUTOREMIX_GPU)
        if (try_gpu) {
            try {
                Ort::SessionOptions gpu_opts;
                gpu_opts.SetGraphOptimizationLevel(GraphOptimizationLevel::ORT_ENABLE_ALL);
                gpu_opts.SetIntraOpNumThreads(1);
    #if defined(_WIN32)
                Ort::ThrowOnError(OrtSessionOptionsAppendExecutionProvider_DML(gpu_opts, 0));
    #elif defined(__linux__)
                OrtCUDAProviderOptions cuda_opts{};
                cuda_opts.device_id = 0;
                gpu_opts.AppendExecutionProvider_CUDA(cuda_opts);
    #elif defined(__APPLE__) && defined(AUTOREMIX_COREML)
                Ort::ThrowOnError(OrtSessionOptionsAppendExecutionProvider_CoreML(gpu_opts, 0));
    #endif
                auto s = Ort::Session(env,
    #ifdef _WIN32
                    model_path.getFullPathName().toWideCharPointer(),
    #else
                    model_path.getFullPathName().toRawUTF8(),
    #endif
                    gpu_opts);
                juce::Logger::writeToLog("[Demucs] GPU EP session — OK");
                return s;
            } catch (const Ort::Exception& e) {
                juce::Logger::writeToLog(
                    juce::String("[Demucs] GPU EP failed (") + e.what()
                    + "), falling back to CPU");
            }
        }
    #else
        juce::ignoreUnused(try_gpu);
    #endif
        Ort::SessionOptions cpu_opts;
        cpu_opts.SetGraphOptimizationLevel(GraphOptimizationLevel::ORT_ENABLE_ALL);
        cpu_opts.SetIntraOpNumThreads(0);
        juce::Logger::writeToLog("[Demucs] CPU EP session");
        return Ort::Session(env,
    #ifdef _WIN32
            model_path.getFullPathName().toWideCharPointer(),
    #else
            model_path.getFullPathName().toRawUTF8(),
    #endif
            cpu_opts);
    };
    Ort::Session session = make_session(true);
    ```

    Update the file header comment line 17:
    "  - CPU EP by default; CUDA (Linux) / DirectML (Windows) via -DAUTOREMIX_GPU=ON."

    Keep everything after the session construction (MemoryInfo, input_shape, allocator,
    chunk loop) exactly as-is — only the session construction changes.

    Avoid: storing GPU/CPU state across calls. make_session() is called once per
    separate_demucs() invocation; the lambda is local, not a class member.
