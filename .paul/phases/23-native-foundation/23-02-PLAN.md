<objective>

## Goal
Port the sidecar's audio-analysis suite to native C++ as a header-only
utility module: BPM, beats, onsets, bars, key, semitone delta, energy
gating, and EBU R128 LUFS normalization. Uses juce::dsp::FFT only — no
external lib (aubio dropped from foundation; we already have what we
need from JUCE + std).

## Purpose
Engines (24), mashup (25), and the algorithmic separator (23-03) all
need these primitives. Today they live in Python (`sidecar/server/remix/
analysis.py` + `base.py`). Native ports unblock all downstream rewrites
in v4.

Match Python semantics function-for-function so v3 vs v4 outputs are
comparable for regression spot-checks during Phase 27.

## Output
- `plugin/include/dsp/NativeAnalysis.h` — header-only utility module
  under `autoremix::dsp::analysis`
- Functions (all take a `const juce::AudioBuffer<float>&` and a sample
  rate, return plain values / `std::vector`):
  * `detect_bpm(audio, sr) -> double`
  * `detect_beats(audio, sr) -> std::vector<double>` (seconds)
  * `detect_onsets(audio, sr, min_gap_ms=80, threshold=0.3) -> std::vector<double>`
  * `detect_bars(audio, sr, beats_per_bar=4) -> std::vector<double>`
  * `detect_energy_gates(audio, sr, threshold_db=-20, hold_ms=50) -> std::vector<uint8_t>`  (sample-aligned mask)
  * `detect_key(audio, sr) -> std::string` (e.g. "C", "Am", "F#")
  * `semitone_delta(key_from, key_to) -> int` (shortest signed semitone path in (-6, +6])
  * `normalize_lufs(buffer, sr, target_lufs=-14) -> void` (in-place; EBU R128)
- Smoke include from `AutoRemixProcessor.cpp` confirming the header
  compiles in a real TU. No call-site yet — engines wire up in Phase 24.

        Returns std::vector<std::vector<float>> [frames][bins].
      - `onset_envelope(const std::vector<std::vector<float>>& mag, hop, sr)
        -> std::vector<float>` — spectral flux: sum of positive differences
        between consecutive magnitude frames. Already half-wave rectified.

    Public:

      inline double detect_bpm(const juce::AudioBuffer<float>& audio, double sr)
      {
        // 1. mono → STFT magnitudes (fft=2048, hop=512)
        // 2. onset_envelope (spectral flux)
        // 3. Autocorrelate the envelope over a lag range corresponding
        //    to 60–200 BPM. lag_samples = 60.0 * sr / (bpm * hop)
        //    so lag range ≈ [sr*60/(200*hop), sr*60/(60*hop)]
        // 4. Pick autocorrelation peak in that range. Convert lag → BPM.
        // 5. Clamp to [60, 200]. Return double.
      }

      inline std::vector<double>
      detect_beats(const juce::AudioBuffer<float>& audio, double sr)
      {
        // 1. detect_bpm to estimate period (seconds_per_beat)
        // 2. onset envelope (same fft/hop as detect_bpm)
        // 3. Phase: find the first strong peak in the envelope; align
        //    subsequent beats at integer multiples of seconds_per_beat
        //    from that phase, snapping each to the nearest local peak
        //    within ±50ms.
        // 4. Return timestamps in seconds.
      }

      inline std::vector<double>
      detect_onsets(const juce::AudioBuffer<float>& audio, double sr,
                    double min_gap_ms = 80.0, double threshold = 0.3)
      {
        // 1. onset envelope
        // 2. Adaptive peak picking: local max where value > threshold * envelope_local_mean
        //    and gap from previous peak >= min_gap_ms
        // 3. Frame index → seconds (idx * hop / sr)
      }

      inline std::vector<double>
      detect_bars(const juce::AudioBuffer<float>& audio, double sr,
                  int beats_per_bar = 4)
      {
        auto beats = detect_beats(audio, sr);
        std::vector<double> bars;
        for (size_t i = 0; i < beats.size(); i += (size_t)beats_per_bar)
            bars.push_back(beats[i]);
        return bars;
      }

    Add a brief documentation block at the top with units + reference
    to the Python originals.

    Avoid: ad-hoc Goertzel/correlation rolls; use juce::dsp::FFT properly.
    Avoid: std::cout/std::cerr — silent on errors, return sensible defaults
    (e.g., bpm=120, empty vector).

      inline std::vector<uint8_t>
      detect_energy_gates(const juce::AudioBuffer<float>& audio, double sr,
                          double threshold_db = -20.0, double hold_ms = 50.0)
      {
        // 1. Mono. Block size = 512 samples (~12ms @ 44.1k).
        // 2. RMS per block.
        // 3. rms_db = 20*log10(rms + 1e-9). frame_mask = rms_db >= threshold_db.
        // 4. Hold: set hold_blocks blocks to true after each true frame.
        // 5. Upsample to sample-level mask via std::vector<uint8_t> of length
        //    audio.getNumSamples() (each block replicates hop times).
        // 6. Pad with last value if short. Return.
      }

      // EBU R128 K-weighting biquads (pre-filter + RLB-weighting). Cookbook
      // coefficients with reference: ITU-R BS.1770-4.
      // Implement as inline helper:
      //   apply_k_weighting(std::vector<float>& mono, double sr) -> void
      //
      // Block-loudness algorithm:
      //   400 ms window, 75% overlap (100 ms hop)
      //   per-block mean square = (1/N) * sum(x^2)
      //   per-block loudness = -0.691 + 10 * log10(sum_channel_weights * mean_square)
      //   absolute gate: drop blocks below -70 LUFS
      //   relative gate: drop blocks below (gated_mean - 10 LUFS)
      //   integrated_loudness = -0.691 + 10*log10(mean of surviving blocks)

      inline void
      normalize_lufs(juce::AudioBuffer<float>& audio, double sr,
                     double target_lufs = -14.0)
      {
        // Try compute integrated LUFS via the algorithm above.
        // If audio too short (< 400ms) or NaN: fall back to peak norm
        //   (target peak = -1 dBFS).
        // Otherwise: gain_db = target_lufs - measured_lufs;
        //            gain   = pow(10, gain_db / 20);
        //            audio.applyGain(gain);
        // Final safety: if peak > -0.5 dBFS, scale down.
      }

    For the K-weighting biquads, use juce::dsp::IIR::Filter with
    coefficients precomputed for 48 kHz (cookbook reference) and an
    inline coefficient-rescale for arbitrary sample rates (the standard
    bilinear transform of the BS.1770 prototype).

    Reference values for 48 kHz from ITU-R BS.1770-4 (hard-coded):

      // Pre-filter (shelving): b0, b1, b2, a1, a2

## Open Questions (revisit during APPLY)

- **Sample-rate flexibility for LUFS**: hard-coding 48 kHz BS.1770
  coefficients is the pragmatic MVP. If during testing we see noticeable
  drift at 44.1 kHz, add the bilinear remapping helper. Public API
  unchanged either way.
- **Beat-tracking quality**: simple peak-snapping at BPM-derived intervals
  is the MVP. If C&S / mashup outputs sound off-beat compared to v3,
  upgrade to dynamic-programming beat tracking (Ellis 2007) later.
- **Performance**: this is all called once per file (offline). No
  real-time constraint. std::vector allocations + inline header
  duplication are acceptable.
