  <action>
    Create the new file, header-only. Namespace `autoremix::dsp::separators`.
    Mark the public function `inline` (multi-TU friendly).

    Includes:
      <juce_audio_basics/juce_audio_basics.h>
      <juce_dsp/juce_dsp.h>
      <algorithm>, <cmath>

    Struct:

      struct NativeStems {
          juce::AudioBuffer<float> vocals;
          juce::AudioBuffer<float> drums;
          juce::AudioBuffer<float> bass;
          juce::AudioBuffer<float> other;
      };

    Private (anonymous-namespace via static inline) helper:

      // 2nd-order Butterworth HP + 2nd-order LP cascade per channel.
      // Approximates the Python `scipy.signal.butter(4, [lo, hi], btype='band')`
      // closely enough for the algorithmic separator's quality target.
      // For freqs at the edges of [0, sr/2], degrades gracefully:
      //   - lo <= 0:  skip the high-pass stage (pass DC through)
      //   - hi >= sr/2 - 1: skip the low-pass stage
      //   - lo >= hi: return zeros
      static inline juce::AudioBuffer<float>
      bandpass_butter(const juce::AudioBuffer<float>& in,
                      double sr, double loHz, double hiHz)
      {
          const double nyq = sr * 0.5;
          juce::AudioBuffer<float> out;
          out.makeCopyOf(in);
          if (loHz >= hiHz) { out.clear(); return out; }

          const int nCh = out.getNumChannels();
          const int nS  = out.getNumSamples();

          if (loHz > 0.0) {
              auto hp = juce::dsp::IIR::Coefficients<float>::makeHighPass(sr, (float) loHz);
              juce::dsp::IIR::Filter<float> filt;
              filt.coefficients = hp;
              for (int c = 0; c < nCh; ++c) {
                  filt.reset();
                  auto* d = out.getWritePointer(c);
                  for (int i = 0; i < nS; ++i) d[i] = filt.processSample(d[i]);
              }
          }
          if (hiHz < nyq - 1.0) {
              auto lp = juce::dsp::IIR::Coefficients<float>::makeLowPass(sr, (float) hiHz);
              juce::dsp::IIR::Filter<float> filt;
              filt.coefficients = lp;
              for (int c = 0; c < nCh; ++c) {
                  filt.reset();
                  auto* d = out.getWritePointer(c);
                  for (int i = 0; i < nS; ++i) d[i] = filt.processSample(d[i]);
              }
          }
          return out;
      }

    Public:

      inline NativeStems
      separate_algorithmic(const juce::AudioBuffer<float>& input, double sr)
      {
          NativeStems out;
          out.bass   = bandpass_butter(input, sr, 0.0,    250.0);
          out.drums  = bandpass_butter(input, sr, 250.0,  2000.0);
          out.vocals = bandpass_butter(input, sr, 2000.0, 6000.0);
          out.other  = bandpass_butter(input, sr, 6000.0, sr * 0.5);
          return out;
      }

    Doc block at the top:
    - Reference: ports sidecar/server/separators/algorithmic_sep.py
    - Band assignments verbatim from Python
    - Trade-off: 2nd-order HP + LP cascade (≈ 4th-order band) vs Python's
      scipy.butter(4, ...) (true 4th-order Butterworth band). Close enough
      for the "low-quality, just-runs" separator.
    - Always-available (no external deps).

    Avoid: relying on juce::dsp::ProcessorChain for an offline filter
    pass — too heavy for this one-shot use. Direct sample-by-sample
    Filter::processSample is fine.

## Open Questions (revisit during APPLY)

- **Filter order**: 2nd-order HP + 2nd-order LP cascade ≈ 4th-order band
  but with slightly different magnitude shape than scipy's
  butter(4, [lo, hi], 'band'). If A/B tests in Phase 27 reveal audible
  drift vs v3, upgrade to higher-order cascaded biquads or use
  `juce::dsp::FilterDesign::designIIRBandpassHighOrderButterworthMethod`.
- **Empty / mono input handling**: spec'd to pass through (no fake
  stereo synthesis). Matches v3 semantics.
- **Per-sample loop vs juce::dsp::ProcessContext block API**: per-sample
  is simpler for one-shot offline use. Block API matters for real-time;
  not our concern here.
