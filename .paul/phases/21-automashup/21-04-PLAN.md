<objective>

## Goal
Make `ScreenMashup` reachable from the UI by adding a "Mashup" button to
`ScreenStemsReady`. Click → opens a file picker → analyses track B → runs a
second separation → populates `ctx_.file_path_b` + `stems_b` +
`detected_bpm_b/key_b` → navigates to `ScreenId::Mashup`.

## Purpose
Closes the loop on Phase 21's user-facing flow. Track A has already been
loaded and separated by the time `ScreenStemsReady` is shown. Adding a single
button here is the cleanest entry point: the user has finished the A side and
is ready to pair it with a second track.

## Output
- New `ScreenContext::start_mashup_flow` callback
- New "Mashup" button on `ScreenStemsReady` (footer, next to "Choose Style >")
- Editor wires the callback to: FileChooser → background thread →
  analyzeFile → separateStems → status updates → navigate to Mashup
</objective>
