<objective>
## Goal
Replace the raw tempo-ratio slider (0.3–2.0) with a target-BPM slider (40–200 BPM),
display detected BPM and key from `AudioBridge::analyzeFile()`, and replace the chop-ms
slider with a chop-beats slider (0.25–16 beats).

## Purpose
After loading a file the user should see its detected BPM and key, set a target BPM
(not an abstract ratio), and set chop length in musical beats.  The analysis block is
provided by 15-01 (`AudioBridge::analyzeFile`).

## Output
- `detected_bpm_` / `detected_key_` member state filled on file load
- Two read-only labels: "120 BPM  •  Am" shown below waveform or in status line
- `tempo_slider_` re-ranged 40–200, display shows BPM values, default = detected BPM
- `chop_slider_` re-ranged 0.25–16 (beats), suffix "beats", converted to ms in onClick_Play
- `loadEngineDefaults` maps preset `tempo_factor` → target BPM using detected source BPM
- `onClick_Play` computes `tempo_factor = bpm_slider / detected_bpm_` and
  `chop_interval_ms = chop_beats * 60000 / detected_bpm_` before building RemixParams
</objective>

        });
