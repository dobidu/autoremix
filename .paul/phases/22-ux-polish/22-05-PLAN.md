<objective>

## Goal
Final polish pass on Phase 22: friendlier copy in mashup combos, replace
the techy `|` separator with a middle dot, give the user a time-estimate
hint during long renders, and reveal the saved WAV in the system file
manager after Save.

## Purpose
None of these alone is a bug — together they remove the last few minor
papercuts surfaced in the Phase 21 UX audit. Phase 22 ends here; v3.2.0
release is the next milestone.

## Output
- ScreenMashup template combo: "Custom" → "Free mix"
- ScreenMashup target key combo: "Anchor to A" → "Match A's key"
- ScreenMashup track headers: `TRACK A | filename` → `TRACK A · filename`
- ScreenMashup rendering overlay hint: append a time-estimate line
- ScreenRender save flow: after copyFileTo succeeds, reveal in file manager
</objective>

         "TRACK B | " → juce::String::fromUTF8("TRACK B \xC2\xB7 ")

    4. In paint() (or the rendering-overlay block), add an additional
       label-positioned drawText call below `rendering_hint_lbl_`
       with text:
         "Algorithmic: ~30s  \xC2\xB7  Demucs ML: 1\xE2\x80\x934 min per track"
       Style: secondary font, AR::COMMENT color with 0.6 alpha.

       Easier alternative: extend rendering_hint_lbl_'s text into 2 lines.
       Use a JUCE multi-line label by setting setMinimumHorizontalScale + 
       setJustificationType(centred). Or split into a second label.

       Recommended: add a NEW member juce::Label rendering_time_lbl_
       (initialized in ctor like rendering_hint_lbl_), set bounds in
       resized() just below the existing hint, and toggle visibility
       in setRendering() in the same loop.
